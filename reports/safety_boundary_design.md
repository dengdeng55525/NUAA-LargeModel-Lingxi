# 安全边界与拒答机制设计

## 设计目标

家庭陪伴机器人面对情绪表达时，不能把自己包装成医生或心理治疗师，也不能在自伤、伤人、诊断、用药等高风险场景下继续普通陪聊。本项目加入轻量安全边界模块，在模型生成前先做规则检测，一旦命中高风险表达，就直接输出谨慎、非诊断、非治疗的支持性回复。

该模块不是医学风险评估系统，只是课程项目中的安全护栏，用于降低陪伴机器人输出不当建议的概率。

## 检测范围

模块位置：`src/lingxi/safety.py`

当前检测以下类别：

| 类别 | 示例关键词 | 处理方式 |
| --- | --- | --- |
| 自伤风险 | 自伤、自杀、伤害自己、不想活、结束生命、撑不下去 | 触发高风险安全回复 |
| 伤人风险 | 伤害别人、报复、想伤人、打死 | 触发高风险安全回复 |
| 极端痛苦 | 极端痛苦、活着没意思、没有希望、彻底崩溃 | 触发高风险安全回复 |
| 长期失眠/健康问题 | 长期失眠、好多天没睡、睡眠障碍 | 触发边界提醒 |
| 用药问题 | 吃药、药量、停药、抗抑郁药、安眠药 | 触发边界提醒 |
| 诊断问题 | 诊断、确诊、抑郁症、焦虑症、双相 | 触发边界提醒 |

## 回复策略

高风险表达不进入普通模型生成流程，而是直接返回固定安全回复：

```text
我能感受到你现在很难受。这个问题不适合只由陪伴机器人判断或处理。建议你尽快联系身边可信任的人，或者寻求专业帮助。如果你现在有伤害自己或伤害别人的冲动，请优先保证安全，并联系当地急救电话或危机干预热线。我可以先陪你做几次缓慢呼吸，或者帮你整理想对身边人或专业人士说的话。
```

涉及诊断、用药、长期失眠等表达时，系统会提醒用户联系专业人士，并提供“整理症状和问题”的支持，而不是给出医学判断。

## 接入方式

当前安全模块已接入 `scripts/memory_chat.py`。执行流程为：

1. 读取用户输入。
2. 调用 `check_safety(user_text)`。
3. 如果触发安全边界，直接输出安全回复，绕过普通陪聊生成流程。
4. 如果没有触发，再进入短期记忆 prompt 构造和模型生成。

查看效果：

```bash
python scripts/memory_chat.py \
  --user "我真的撑不下去了，觉得消失了也没关系。" \
  --no-save
```

涉及诊断或用药的例子：

```bash
python scripts/memory_chat.py \
  --user "我是不是抑郁症，要不要吃药？" \
  --no-save
```

如需调试原始模型行为，可显式关闭安全模块：

```bash
python scripts/memory_chat.py \
  --disable-safety \
  --dry-run \
  --user "我是不是抑郁症，要不要吃药？"
```

## 报告表述

为了避免陪伴机器人在高风险情绪场景下输出不当建议，本项目设计了安全边界模块，对自伤、伤人、诊断、用药、长期失眠等高风险表达进行识别，并引导系统采用非诊断、非治疗、支持性回应。该模块位于模型推理之前，属于规则护栏，与 LoRA 微调模型解耦，因此即使模型生成能力不足，也能对关键安全场景提供稳定兜底。
